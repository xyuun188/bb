# 模型持续训练与模拟交易闭环整改计划

日期：2026-07-22

项目：`E:\code\bb`

状态：待执行

实施范围：模拟盘、模型训练、策略选择、分析调度、训练看板

暂不实施：实盘下单逻辑改造

## 1. 文档定位

本计划整合以下问题的整改方向：

- 模型长期不能晋级，导致无法真正参与模拟交易。
- 没有交易就缺少真实成交、手续费、滑点、持仓和平仓样本。
- 当前训练输入虽然是多维度的，但模型的输出和实际权限仍偏向做多、做空方向。
- 币种、持仓周期、杠杆、仓位和平仓没有形成统一的模型决策闭环。
- 训练数据的盈亏、部分平仓、手续费和持仓结果必须先证明正确。
- 模型和策略的训练结果、当前表现及其实际影响无法在系统中清楚查看。
- 当前系统存在重复、无实际用途、过度技术化或用户难以理解的数据展示。
- 市场分析吞吐不足，部分币种被频繁分析，其他币种长期得不到覆盖。
- 模拟盘应该允许受控探索，但不能通过无规则乱开仓制造低质量样本。

本计划替代旧计划中“模型或策略只有通过一次性晋级门槛后才能影响模拟交易”的执行方式。旧计划中的资金安全、费后收益、最大亏损、禁止重复下单、数据可信和风险否决原则继续保留。

## 2. 最终目标

将系统改造成持续学习的模拟交易闭环：

```text
全市场数据
  -> 候选币公平轮换与优势筛选
  -> 所有模型持续分析
  -> 统一决策层综合模型意见
  -> 生成完整交易方案
  -> 风险系统裁剪杠杆、仓位和最大亏损
  -> 模拟盘执行与持仓管理
  -> 交易账本核对
  -> 可信真实样本与影子样本
  -> 持续训练、验证和动态调整模型权重
  -> 训练情况与决策过程在看板中完整展示
```

目标不是保证训练时间越长就一定盈利，而是确保：

1. 错误数据不能进入训练。
2. 不同决策环节的好坏能够分别归因。
3. 所有模型持续参与分析和学习，不再等待一次性晋级。
4. 模型近期表现决定其交易影响力，而不是决定其是否有资格继续训练。
5. 偶然规律不能因为训练集成绩好就获得较大交易影响力。
6. 市场规律变化后，失效模型能够自动降权并继续学习。
7. 每项结果都能从看板追溯到对应分析、交易、行情和训练样本。

## 3. 不可突破的边界

- 本阶段只改模拟盘，不改变实盘账户和实盘下单逻辑。
- 所有模型参与分析，不等于所有模型拥有相同下单权重。
- 模型可以建议杠杆和仓位，但不能绕过风险系统直接执行。
- 风险系统始终保留拒绝交易、降低杠杆和缩小仓位的权力。
- 不放宽最大亏损、重复下单、账户一致性、合约规则和流动性限制。
- 不以胜率作为主要训练目标或策略选择目标。
- 不用强制每轮开仓的方式制造训练样本。
- 不把训练集内部成绩当成模型已经有效的证明。
- 不把缺少成交、费用、剩余仓位或时间证据的数据用于正式训练。
- 不因为某个模型近期表现差就删除其训练能力；应降低影响力并继续观察。

## 4. 统一评价目标

模型、策略和完整交易方案统一按以下顺序评价：

1. 扣除手续费、滑点和资金费后的累计总收益。
2. 盈利总额与亏损总额的比例。
3. 最大回撤和连续亏损程度。
4. 极端行情下的尾部亏损。
5. 收益在不同时间、币种、方向和行情状态下的稳定性。
6. 预期收益与实际收益是否校准一致。
7. 执行成本是否吞噬了预测优势。

胜率只作为辅助说明。低胜率但费后总收益稳定为正的模型或策略可以获得较高权重；高胜率但偶发巨亏导致总收益为负的模型或策略必须降权。

## 5. 目标决策结构

### 5.1 所有模型持续参与

所有可用模型在每轮分析中持续输出各自擅长的证据：

- 趋势与动量判断。
- 时间序列和不同持仓周期预测。
- 成交量、盘口与流动性分析。
- 衍生品、资金费率、持仓量和清算风险。
- 新闻、社交和市场情绪修正。
- 盈利概率、亏损范围和极端风险分析。
- 持仓继续、部分退出和全部退出分析。

所有输出都保存并参与后续训练。持续参与指持续分析和提供证据，不表示每个模型都能独立下单。

### 5.2 统一决策层

只能由一个统一决策层综合模型意见并生成最终模拟交易建议，禁止多个模型各自创建重复订单。

模型权重必须细分到：

- 当前币种。
- 做多或做空方向。
- 趋势、震荡或极端波动状态。
- 目标持仓周期。
- 最近一段时间的费后实际表现。

不能简单按模型数量投票，也不能让多个角色包装下的同一个模型被重复计算为多份独立证据。

### 5.3 完整交易方案

每次最终建议必须包含：

- 是否交易，允许明确选择不交易。
- 推荐币种及候选币排序。
- 做多或做空。
- 建议入场价格范围和建议有效时间。
- 预计持仓周期和最大持仓时间。
- 提前失效、继续持有、部分平仓和全部平仓条件。
- 扣除费用后的预期收益范围。
- 正常亏损范围和极端亏损范围。
- 杠杆建议。
- 仓位金额或账户资金占比建议。
- 判断可信程度、支持理由和反对理由。

模型原始建议、统一决策结果、风险系统调整和最终执行结果必须分别保存，防止错误归因。

## 6. 模型与策略的持续治理

### 6.1 取消硬性晋级

取消“没有晋级就完全不能参与模拟交易”的二元门槛，改为连续权重：

- 新模型从很小的影响力开始。
- 近期费后表现稳定改善时逐步提高影响力。
- 表现恶化时自动降低影响力。
- 严重失效时可以接近零交易影响力，但仍继续分析、训练和生成影子结果。
- 重新恢复有效后可以自动恢复影响力。

这里的权重变化不是另一种名称的人工晋级，而是持续、可回退、由最新证据驱动的自动调整。

### 6.2 策略动态选择

不永久使用一套历史成绩最好的策略：

- 同时维护趋势、震荡、反转等适用于不同市场状态的候选策略。
- 每种市场状态选择一个主要策略。
- 其他策略以小仓位或影子方式持续验证。
- 根据近期费后总收益、回撤、极端亏损和稳定性调整策略权重。
- 市场状态变化时允许主要策略自动切换。
- 保留最近稳定版本用于回退，但清除失效的重复分支和无人使用的旧执行代码。

## 7. 分阶段整改与验收

每个阶段都执行同一闭环：实施 -> 最小相关测试 -> 数据核对 -> 问题修复 -> 再次测试。只有该阶段所有验收项通过后，才进入下一阶段。

### 阶段一：建立唯一可信的交易账本

整改内容：

- 统一做多、做空、加仓、部分平仓和全部平仓的数量及成本计算。
- 正确记录每次实际成交均价、手续费、资金费和滑点。
- 正确区分已实现盈亏、未实现盈亏和账户权益变化。
- 部分平仓后必须保留正确的剩余仓位和剩余成本。
- 模拟盘和实盘数据完全隔离，但复用相同的财务计算规则。
- 交易所保护单证据不可用时，明确区分“订单不存在”和“证据查询失败”。

自检与验收：

- 固定价格的做多、做空、加仓和部分平仓案例全部可以人工复算。
- 每笔交易的账户变化能够与费后盈亏对应到系统精度允许的最小误差。
- 历史成交、当前持仓和账户权益三者一致。
- 部分平仓后不会把剩余仓位错误显示为已全部结束。
- 数据缺失或核对失败时自动隔离，不进入训练。

### 阶段二：重建可信训练数据

整改内容：

- 每次市场分析保存当时可见的行情、模型建议和决策上下文。
- 在10、30、60分钟等多个时间范围成熟后生成结果。
- 同时计算如果做多、做空和不交易的结果。
- 记录最大有利波动、最大不利波动和不同退出时间的结果。
- 计算不同杠杆下的影子收益、回撤、费用和清算风险。
- 真实交易用于学习成交、费用、滑点和执行偏差。
- 未下单机会继续生成影子结果，用于学习币种、方向和持仓时间。

自检与验收：

- 未结束、重复、时间错位和费用不完整的样本无法进入可信训练集。
- 每条可信样本都能追溯到原始行情、分析和交易证据。
- 训练只能使用决策发生前已经存在的信息。
- 训练集和考试集严格按时间分离。
- 样本数量能够按币种、方向、行情状态和持仓周期准确汇总。

### 阶段三：实现多维完整建议

整改内容：

- 将只偏向方向的输出扩展为完整交易方案。
- 分别记录币种、方向、入场、持仓时间、平仓、杠杆和仓位建议。
- 建议必须包含扣费后收益、亏损范围和极端风险。
- 模型建议与风险系统最终裁剪结果分别记录。
- 持仓期间持续重新评估，不把开仓时的预测永久固定。

自检与验收：

- 每笔下单都有完整交易方案，没有退出计划则不能下单。
- 每项模型建议都能与实际结果分别比较。
- 能够识别“方向正确但退出错误”“策略正确但费用吞噬收益”等不同原因。
- 不再把整笔交易的最终亏损简单归咎于方向模型。

### 阶段四：实现模型持续参与和动态权重

整改内容：

- 所有健康模型每轮持续提供分析。
- 根据近期费后实际结果和影子结果计算分场景权重。
- 新模型从低影响力开始，表现改善后平滑增加。
- 失效模型自动降权，但继续训练和观察。
- 保存每次权重变化的证据和原因。

自检与验收：

- 不再因为没有模型晋级而完全停止模拟交易。
- 同一个模型的多个角色不会被重复当成独立投票。
- 表现差的模型不能靠数量压过表现好的模型。
- 权重变化可解释、可回退，并能从历史结果复算。

### 阶段五：实现策略持续生成和动态路由

整改内容：

- 模型能够生成适用于不同市场状态的候选策略。
- 对候选策略进行严格的时间外验证和逐币验证。
- 为不同市场状态选择主要策略和挑战策略。
- 持续根据最新费后表现调整策略权重。

自检与验收：

- 策略不能只依赖某一个币种或某一小段行情。
- 训练数据成绩好但未来时间成绩差的策略不能获得主要权重。
- 市场状态改变后能够自动切换适合的策略。
- 旧稳定策略能够回退，但不会与新策略重复执行订单。

### 阶段六：合并正常模拟交易与训练采样

整改内容：

- 取消“正常订单”和“训练采样订单”的业务分裂。
- 每笔模拟交易既是正常交易，也是潜在训练样本。
- 初期同时保留证据最优交易和极小仓位探索交易。
- 探索覆盖不同币种、方向和持仓周期，不做无规则随机下单。
- 随着可靠证据增加逐步降低探索比例。
- 使用影子计算探索多个杠杆，不通过重复实际订单制造样本。

自检与验收：

- 所有模拟交易进入同一持仓、盈亏和历史记录体系。
- 不会因为训练标记提前平仓、固定一倍杠杆或绕过正常策略。
- 不产生相同币种和方向的重复订单。
- 探索交易的风险、数量和产生原因能够完整追溯。

### 阶段七：完善杠杆、仓位和退出闭环

整改内容：

- 模型学习并建议持仓周期、杠杆和仓位，而不只学习方向。
- 风险系统根据波动、流动性、相关持仓、可用资金和最大亏损裁剪建议。
- 持仓期间持续比较继续持有、部分平仓、全部平仓和替换机会。
- 记录模型建议与风险裁剪造成的结果差异。

自检与验收：

- 模型无法取消硬止损或突破账户最大亏损。
- 杠杆和仓位建议存在清楚的收益与风险依据。
- 部分平仓比例、剩余仓位和后续训练标签保持一致。
- 平仓评价使用费后结果，不因短期浮亏机械退出。

### 阶段八：解决分析吞吐和币种覆盖

整改内容：

- 提前计算并缓存高成本行情指标。
- 不同模型并行分析，单个模型超时不能拖死整轮。
- 市场分析与持仓分析使用独立队列，持仓分析优先。
- 候选币采用优势排序与公平轮换结合。
- 未完成候选自动顺延，但不能被连续跳过。
- 对指标准备、模型分析、行情接口和整轮耗时分别监控。

自检与验收：

- 每个半小时都能看到持续分析活动。
- 所有合格币种都能在规定覆盖窗口内得到分析。
- 不再长期只集中分析少数币种。
- 持仓分析不会被市场候选分析挤占。
- 超时原因和未完成候选能够准确统计。

### 阶段九：优化主面板以外的展示并建设模型训练看板

现有主面板不在展示整改范围内。不得调整主面板的布局、指标、卡片、入口、文字、交互、数据口径或展示逻辑，也不得以本计划为理由重构主面板。新增模型训练看板应作为独立页面接入，不能挤占或改写现有主面板内容。

在增加训练看板之前，只对主面板以外的现有页面、标签页、表格、指标、状态文字和操作入口进行信息审计。

#### 全系统展示清理原则

所有现有展示内容必须归入以下四类，不能继续无规则堆积：

1. 日常必看：直接影响对应页面业务判断的内容，保留在该页面主要区域。
2. 按需查看：有排查价值但不是日常决策必需的内容，收进展开详情或高级诊断页面。
3. 后台审计：模型内部原始分数、内部编号、技术调用过程和原始响应，只保留在日志或审计记录中。
4. 确认无用：没有任何计算、决策、审计或排障用途，且确认没有调用后删除展示和对应废弃代码。

不能仅凭“当前看不懂”就删除底层证据。对交易追溯、训练归因和风险审计仍有价值的数据，应从对应业务页面移到展开详情或高级诊断，并使用清楚中文解释，而不是破坏数据来源。

#### 页面结构整改

- 主面板：保持现状，本阶段不做任何展示修改。
- 持仓：只展示当前真实存在的持仓及持仓期间分析。
- 历史交易：展示已全部结束的交易、部分平仓明细和最终费后结果。
- 市场分析：展示未持仓币种的市场机会和未交易原因。
- 持仓分析：独立展示现有持仓的继续持有、减仓或退出判断。
- 模型训练：展示训练数据质量、模型表现、影响力和变化原因。
- 策略：展示当前主要策略、候选策略、适用行情和实际表现。
- 高级诊断：集中放置接口耗时、原始证据、内部编号和技术排障信息。

市场分析和持仓分析必须在数据来源、查询条件、页面标签和空状态上明确分开，不能因为市场分析有数据就让用户误以为持仓分析也在工作。

#### 文字与指标整改

- 页面统一使用自然、明确的中文，不直接向用户展示内部字段名或函数名。
- 技术状态必须翻译成用户能采取行动的描述，例如明确说明“保护单查询失败”“保护单不存在”或“正在重试”。
- 相同含义只保留一个主指标，删除重复统计和互相矛盾的口径。
- 所有盈亏默认展示扣除手续费、滑点和资金费后的结果，并清楚标明口径。
- 百分比、金额、杠杆、数量和时间统一精度与单位。
- 不使用含义不明的缩写、内部等级、原始评分或没有解释的状态码。
- 必须保留的专业概念通过简短中文说明或悬停提示解释，不在业务页面堆积教程文字。
- 无数据、加载中、查询失败、服务停止和确实为零必须显示为不同状态。
- 模拟盘与实盘切换后，持仓、历史、盈亏和账户数据必须整体切换，不能混合。

#### 操作与视觉整改

- 高频查看内容优先，低频诊断内容后置。
- 表格默认展示最重要列，次要列通过详情查看。
- 操作按钮使用明确动作和熟悉图标，危险操作必须二次确认。
- 减少无意义卡片、装饰和大面积空白，保持适合持续查看的紧凑信息密度。
- 避免卡片嵌套、重复标题、重复状态和同一信息多处不一致。
- 桌面和移动端均保证文字、表格、筛选器、标签页和按钮不重叠。
- 颜色只用于表达盈亏、风险、状态和选中关系，不能只靠颜色传达含义。

#### 展示清理自检与验收

- 为每个被删除或移动的展示项记录保留、折叠、后台保留或删除的理由。
- 删除展示前检查是否仍被交易、训练、风控、接口或审计使用。
- 主面板修改文件清单必须为空；截图和关键页面契约测试证明整改前后主面板保持不变。
- 模拟盘与实盘切换后所有相关数据来源一致。
- 市场分析与持仓分析互不混淆，并分别显示最近更新时间和运行状态。
- 所有关键页面完成桌面和移动端截图检查以及真实交互验证。

#### 模型训练看板

看板必须直接回答“训练到了什么程度、是否在变好、当前如何影响交易”。

#### 总览

- 最近一次训练时间、结果、耗时和下一次计划。
- 今日新增、已成熟、可信、隔离和等待结果的样本数量。
- 训练是否正在运行、停止或失败及其原因。
- 当前模型整体对模拟交易的影响程度。

#### 数据质量

- 按币种、方向、行情状态和持仓周期展示样本分布。
- 展示手续费、滑点、资金费和成交证据完整率。
- 展示重复、时间错位、部分平仓不一致和账本核对失败数量。
- 所有异常数量可以继续查看对应样本和交易。

#### 单个模型

- 模型负责的能力和最近训练所用样本数量。
- 近期影响力及升降原因。
- 训练阶段和未来时间考试阶段的费后总收益。
- 盈利亏损比、最大回撤、极端亏损和成本占比。
- 在不同币种、方向、行情状态和持仓周期下的表现。
- 预期收益与实际收益的偏差趋势。

#### 策略情况

- 当前主要策略、挑战策略和适用市场状态。
- 每套策略的真实模拟结果和影子结果。
- 策略权重变化、变化原因和历史版本。

#### 实时决策追踪

- 本轮分析了哪些币种。
- 每个模型支持或反对什么。
- 统一决策层采用了哪些证据。
- 风险系统调整了哪些杠杆、仓位和退出条件。
- 最终为什么下单或不下单。
- 交易结束后各决策环节分别表现如何。

#### 告警

- 训练失败或长期未运行。
- 数据核对失败或可信样本突然下降。
- 模型近期明显失效。
- 策略回撤超限。
- 分析轮次超时或币种覆盖不足。
- 建议与实际执行长期偏离。

自检与验收：

- 页面所有汇总数字与训练数据、交易账本和分析记录一致。
- 每个汇总数字能够追溯到具体记录。
- 不用单一“训练进度百分比”掩盖真实表现。
- 页面同时展示训练成绩与未来时间考试成绩。
- 桌面和移动端均无文字、表格和控制项重叠。

### 阶段十：整体验收、清理与持续观察

整改内容：

- 删除已经被新闭环替代且确认无调用的旧晋级分支、重复采样分支和无效兼容代码。
- 保留必要的数据迁移、稳定版本和回退能力。
- 执行完整测试、模拟盘线上同步和持续运行观察。
- 核对服务状态、分析频率、交易链路、训练任务和看板数据。

最终技术验收：

- 无重复下单，模拟盘与实盘数据不混用。
- 部分平仓后剩余持仓和历史记录准确。
- 每条可信训练数据都可追溯且计算正确。
- 市场分析和持仓分析持续运行。
- 所有合格币种得到公平覆盖。
- 每笔交易可还原模型建议、综合决策、风控调整、实际成交和最终结果。
- 模型表现恶化后自动降权，恢复有效后能够重新提高。
- 模型、策略、训练和交易看板数据与底层事实一致。
- 相关测试、线上同步、运行验证、GitHub提交推送和Hindsight记录全部完成。

## 8. 实施顺序与停止条件

必须严格按阶段一至阶段十顺序执行。以下情况出现时停止进入下一阶段，并继续修复当前阶段：

- 交易盈亏无法人工复算。
- 账户、持仓和历史成交无法核对。
- 不完整样本仍能进入训练。
- 发现未来数据泄漏。
- 模型建议和实际执行无法区分。
- 动态权重无法解释或无法回退。
- 重复下单、仓位消失或部分平仓错误再次出现。
- 看板数字与底层数据不一致。
- 模拟盘服务运行异常或分析任务中断。

## 9. 完成定义

本计划存在两个不同的完成标准：

### 工程整改完成

所有阶段测试和验收通过，模拟盘形成可持续、可追溯、可回退的多维训练与交易闭环，系统能够清楚展示模型训练情况及其对交易的实际影响。

### 盈利能力得到证明

必须通过足够长时间、未参与训练的未来模拟交易数据证明费后累计收益、回撤和极端亏损达到预设标准。代码完成、训练集盈利或短期偶然盈利都不能宣布模型已经具备稳定盈利能力。

工程整改可以解决“没有晋级就不交易、不交易就没有有效样本”的死循环，也能降低模型长期学习错误数据的风险，但不对未来盈利作无证据保证。
